# Contributing

AI Freedom Trust Federation welcomes practical, accountable contributions that improve clarity, safety, usefulness, and maintainability.

## Contribution Standards

- Keep public language grounded and understandable.
- Respect human governance, consent, and user agency.
- Prefer working systems, clear documentation, and testable improvements over hype.
- Keep repository boundaries clear; do not mix unrelated project concerns.
- Avoid unsupported claims about security, finance, health, legal status, decentralization, or AI capability.

## Before Opening a Pull Request

1. Read the repository README and any local `AGENTS.md`.
2. Keep changes scoped to one clear purpose.
3. Run the repository's documented checks when available.
4. Update docs when behavior, commands, structure, or public messaging changes.
5. Explain the risk level and verification performed.

## Repository Status

Each repository should declare one status in its README:

- `Active product`
- `Infrastructure`
- `Federation core`
- `Research`
- `Incubating`
- `Legacy`
- `Fork`
